简介

  • go语言单元测试框架，解决go语言自带的测试框架go test相关不足的。

特点

  • 详细看官方描述，如果有学习过c语言的话，有点类似c语言中的CUnit的测试框架结构，CUnit官方

实战

安装

go get -u gopkg.in/check.v1

使用

API 文档

gocheck 检查包的 API文档可在线获得:

  • https://gopkg.in/check.v1

官方demo

package calc

import (
	"io"
	"testing"

	. "gopkg.in/check.v1"
)

// Hook up go check into the "go test" runner.
func Test(t *testing.T) { TestingT(t) }

type MySuite struct{}

var _ = Suite(&MySuite{})

func (s *MySuite) TestHelloWorld(c *C) {
	c.Assert(42, Equals, "42")
	c.Assert(io.ErrClosedPipe, ErrorMatches, "io: .*on closed pipe")
	c.Check(42, Equals, 42)
}

相关知识点说明

特殊的函数

  • func (s *SuiteType) SetUpSuite(c *C)

  • 当套件开始运行时，运行一次。

  • func (s *SuiteType) SetUpTest(c *C)

    • 在每个测试或基准开始运行之前运行。

  • func (s *SuiteType) TearDownTest(c *C)

    • 每次测试或基准测试运行后运行。

  • func (s *SuiteType) TearDownSuite(c *C)

    • 在所有测试或基准测试完成运行后运行一次

  • Demo

源文件:

package calc

func add(a, b int) int {
	return a + b
}

测试文件:

package calc

import (
	"testing"

	. "gopkg.in/check.v1"
)

// Hook up go check into the "go test" runner.
func Test(t *testing.T) { TestingT(t) }

type MySuite struct{}

var _ = Suite(&MySuite{})

func (s *MySuite) TestCalc(c *C) {
	c.Assert(3, Equals, add(1, 2))
	c.Check(3, Equals, add(1, 2))
}

func (s *MySuite) SetUpSuite(c *C) {
	c.Log("SetUpSuite")
}

func (s *MySuite) SetUpTest(c *C) {
	c.Log("SetUpTest")
}

func (s *MySuite) TearDownTest(c *C) {
	c.Log("TearDownTest")
}

func (s *MySuite) TearDownSuite(c *C) {
	c.Log("TearDownSuite")
}

		c.Assert(1, Equals, 1)

跳过测试

Assert and Check

断言和检查，接收相同的参数。

区别： Assert失败的时，测试会立即中断，而 Check将失败测试，返回 false，并允许它继续进行进一步检查。

断言和检查具有以下类型：

func (c *C) Assert(obtained interface{}, chk Checker, ...args interface{})
func (c *C) Check(obtained interface{}, chk Checker, ...args interface{}) bool

选择要运行的测试

gocheck 可以根据测试名称、套件名称或两者筛选出测试。

运行 go 测试 时提供命令行选项*- check.f*

eg:

$ go test -check.f MyTestSuite
$ go test -check.f "Test.*Works"
$ go test -check.f "MyTestSuite.Test.*Works"

详细模式

gocheck 通过 - check. v 和 - check. vv 标志提供两个级别的冗长。

在第一种模式下，还将报告通过测试。

第二种模式将完全禁用日志缓存，并将流开始和结束套件调用以及直接登录到输出之间的所有内容。例如，这对于调试挂起测试很有用。

总结

  • 目前开发使用gocheck比较少，之所以学习这个框架是因为goland IDE只要你go get gocheck的情况，可以有 gocheck的支持，右键项目->run->gocheck。
  • 目前使用比较多的测试框架也是基于 go test 的 goconvey，支持web显示和覆盖率，表示很喜欢。后续也会更新一个goconvey的详细使用方法。
